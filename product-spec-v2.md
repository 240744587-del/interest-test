## 2. 产品定义

### 2.1 一句话描述

「探路者」是一款按教育阶段提供不同题目的兴趣与方向探索工具，通过结构化答题、规则评分和 AI 辅助解读，帮助用户认识当前的兴趣、能力倾向、驱动力和探索方向。

### 2.2 产品边界

本产品是自我探索和职业启蒙工具，不是：

- 心理诊断或医疗工具；
- 经过常模验证的专业心理量表；
- 人才选拔、升学录取或就业筛选依据；
- 对专业、职业或人生方向的确定性判断。

产品界面和报告不得使用“诊断”“科学量表”“精准测出”“最适合的职业”“天生注定”等表述。

推荐表述包括：

- “当前倾向”
- “兴趣种子”
- “值得进一步体验的方向”
- “建议通过真实活动继续验证”

## 3. 用户分层

层级按用户当前教育阶段划分，不根据具体年龄自动推断。

| 层级 | 用户范围 | 核心任务 | 报告重点 |
|------|----------|----------|----------|
| L1 | 小学生 | 发现兴趣种子和能力苗头 | 活动体验、兴趣培育和家长支持 |
| L2 | 初中生 | 认识兴趣和能力倾向 | 社团、学科和体验方向 |
| L3 | 高中生 | 探索专业与发展方向 | 选科、专业领域和验证行动 |
| L4 | 大学及以上学生、已工作成人 | 检视学习或职业方向 | 专业发展、工作环境和转型探索 |

首页只要求用户选择上述教育阶段，不收集学校、年级、生日或年龄。

## 4. 测评框架

### 4.1 六个维度

| 维度 | 名称 | 主要输出 |
|------|------|----------|
| D1 | 心理能量模式 | 能量方向、心流触发、能量消耗和刺激偏好 |
| D2 | 天然能力图谱 | 8 种多元智能的相对倾向 |
| D3 | RIASEC 兴趣图谱 | R、I、A、S、E、C 六类兴趣及高级指标 |
| D4 | 核心驱动力 | 创造、探索、连接、影响、自由、安全和平衡 |
| D5 | 认知与决策风格 | E-I、S-N、T-F、J-P 连续光谱 |
| D6 | 职业发展准备度 | 自我认知、职业信息、决策和现实取向 |

L1 不测 D6。L2 只测 D6 的自我认知部分。L3 和 L4 按对应题库测量 D6。

### 4.2 题目类型

MVP 支持以下五种计分题型：

1. 二选一；
2. 场景选择；
3. 1–5 分量表；
4. 排序；
5. 迫选，选择最喜欢和最不喜欢的选项。

MVP 不设置开放文本题，不允许用户输入自由文本。

### 5.1 题量

| 层级 | v2.0 原汇总 | 删除开放题 | MVP 最终题量 |
|------|-------------|------------|--------------|
| L1 | 32 | 0 | 32 |
| L2 | 46 | 0 | 46 |
| L3 | 55 | 2 | 53 |
| L4 | 62 | 3 | 59 |

### 5.2 合并规则

- L1：用 6 道荣格题替换原 Q27–Q30，其余题目保留，最终 32 题。
- L2：用 8 道荣格题替换原 Q35–Q40，增加 4 道准备度题，最终 46 题。
- L3：用 8 道荣格题替换原维度 5 的 6 道题，增加 8 道准备度题；4 道 RIASEC 迫选题替换 4 道原 RIASEC 题，不额外增加题量；删除 2 道开放题，最终 53 题。
- L4：用 8 道荣格题替换原维度 5 的 7 道题，增加 10 道准备度题；前三道 RIASEC 迫选题替换 3 道原 RIASEC 题，第四道作为新增题；删除 3 道开放题，最终 59 题。

题库数据化时，应优先替换测量目标重复、区分度较弱或社会赞许性较强的原 RIASEC 题。被替换题目的具体 ID 必须记录在题库清单中，并通过题量和维度覆盖检查后再进入页面开发。

数据化时必须为所有题目分配稳定且唯一的 ID，不沿用不同文档中可能重复的局部编号。

推荐格式：

```text
L1-ENERGY-001
L2-JUNG-001
L3-READY-001
L4-RIASEC-FC-001
```

## 6. 评分规则

### 6.1 原始计分

- 二选一：选中项对应子维度 `+2`。
- 场景选择：按题目选项标注计分，通常为 `+3`。
- 量表题：1–5 分直接计入对应子维度；反向题在题库数据中显式标记并反向计分。
- 排序题：第一名 `+5`，第二名 `+3`，第三名 `+1`，其余 `0`。
- RIASEC 迫选：最喜欢 `+4`，最不喜欢 `-2`；单选迫选题按题目标注计分。

### 6.2 标准化得分

MVP 没有样本常模。所有展示分数均为理论量程内的标准化得分，不是人群百分位。

每个子维度按以下公式转换：

```text
标准化得分 = (原始分 - 理论最低分) / (理论最高分 - 理论最低分) × 100
```

结果限制在 `0–100`，展示时取整数。若一个子维度理论最高分等于理论最低分，则题库完整性检查必须失败，不生成报告。

界面统一使用“得分”或“倾向强度”，不得使用“百分位”“超过了多少人”或“人群排名”。

### 6.3 双向光谱

E-I、S-N、T-F、J-P 采用同一条 0–100 光谱展示：

- `0` 表示完全偏向左侧；
- `50` 表示两侧相对平衡；
- `100` 表示完全偏向右侧。

光谱值根据两端标准化得分的相对占比计算。两端均无有效得分时，题库完整性检查必须失败。

### 6.4 RIASEC 高级指标

- 兴趣代码：按六类标准化得分从高到低取前三位。
- 一致性：根据前两类在 RIASEC 六角模型中的距离分为高、中、低。
- 区分度：根据最高分与最低分的标准化分差分为高、中、低。
- 明确度：根据六类得分的集中程度分为高、中、低。

具体阈值在评分引擎实现前通过固定规则定义并写入自动化测试。MVP 不根据用户样本动态调整阈值。

### 6.5 一致性提示

一致性检测只用于提示结果可信度，不使用“测谎”措辞，也不阻止用户查看报告。

- 配对题明显矛盾时，报告显示“本次回答中有部分倾向不够稳定，建议结合真实体验继续观察”。
- 不以一致性结果推断用户故意作假。

## 7. 报告规则

### 7.1 固定算法输出

以下内容由本地评分和映射规则生成，不依赖 AI：

- 六个维度及子维度得分；
- 荣格四组连续光谱；
- RIASEC 六类得分、兴趣代码及高级指标；
- Top 能力、核心驱动力和能量倾向；
- 与层级匹配的候选探索领域。

### 7.2 AI 输入边界

AI 只接收：

- 层级 `L1`、`L2`、`L3` 或 `L4`；
- 各维度及子维度的标准化汇总分；
- RIASEC 兴趣代码和高级指标；
- 一致性提示标记；
- 本地规则选出的候选探索领域。

AI 不接收：

- 用户身份信息；
- 逐题答案；
- 题目作答时间；
- 设备、位置或网络标识；
- 任何自由文本。

### 7.3 AI 输出约束

AI 负责把结构化结果改写成年龄适配的说明，不得自行改变分数或制造新的测评结论。

报告必须：

- 使用概率性、发展性语言；
- 说明结果反映的是当前回答倾向；
- 鼓励通过真实活动验证；
- 对 L1–L3 避免确定性职业推荐；
- 明确产品不构成心理诊断、升学或职业决策意见。

报告不得：

- 输出人格诊断、心理疾病或健康判断；
- 推断家庭关系、经济状况、智商或学习成绩；
- 宣称用户只能或必须选择某个方向；
- 根据教育阶段之外的信息猜测年龄、性别或身份。

### 7.4 AI 失败降级

AI 调用失败、超时或返回内容不符合结构要求时，系统使用本地模板展示评分结果和探索建议。用户不需要重新答题。

## 8. 数据和隐私

### 8.1 数据最小化

MVP 不收集：

- 姓名、昵称、性别；
- 年龄、生日、身份证明；
- 学校、年级、单位；
- 手机号、邮箱或社交账号；
- 地理位置；
- 开放文本；
- 用于广告画像或跨站追踪的标识。

唯一由用户主动选择的背景信息是教育阶段。

### 8.2 不存储原则

- 不设置账号系统。
- 不将逐题答案、评分结果或报告写入数据库、文件或对象存储。
- 不使用浏览器持久化存储保存答案或报告。
- 答案仅在完成当次评分所需的内存和请求生命周期中存在。
- 报告页面关闭或刷新后，不保证恢复。
- 不建立样本库，不以训练、研究、常模建设或产品优化为由留存答题数据。

### 8.3 日志与统计

- 服务端日志不得记录请求正文、逐题答案、评分结果或 AI 提示词正文。
- 错误日志只记录错误类别、时间和不含用户内容的请求标识。
- MVP 不接入记录答题内容、用户画像或报告内容的分析工具。
- 如使用基础访问统计，只允许记录页面级访问和匿名汇总事件，并在上线前单独评估。

### 8.4 用户告知

开始答题前必须用简明语言说明：

1. 这是兴趣探索工具，不是专业诊断或决策依据；
2. 不要求填写姓名、学校等身份信息；
3. 答案只用于当次评分，不建立个人档案或样本库；
4. 匿名汇总分会发送给第三方 AI 服务生成文字解读；
5. 用户可以拒绝 AI 解读，并仅查看本地规则生成的基础报告。

L1 和 L2 入口需要提示由监护人知情后使用。正式公开上线前，应对未成年人使用流程、第三方 AI 数据处理和隐私文本进行专项合规评估。

## 9. 核心数据契约

### 9.1 答案提交

```typescript
type Level = 'L1' | 'L2' | 'L3' | 'L4';

type AnswerValue =
  | { kind: 'single'; optionId: string }
  | { kind: 'likert'; value: 1 | 2 | 3 | 4 | 5 }
  | { kind: 'ranking'; optionIds: string[] }
  | { kind: 'forcedChoice'; mostOptionId: string; leastOptionId?: string };

interface Answer {
  questionId: string;
  value: AnswerValue;
}

interface ScoreRequest {
  level: Level;
  answers: Answer[];
}
```

`ScoreRequest` 不包含年龄、身份信息、开始时间、完成时间或开放文本。

### 9.2 评分结果

```typescript
interface NormalizedSubScore {
  key: string;
  label: string;
  score: number;
}

interface DimensionResult {
  key: string;
  label: string;
  subScores: NormalizedSubScore[];
}

interface ScoreResult {
  level: Level;
  dimensions: DimensionResult[];
  jung: { EI: number; SN: number; TF: number; JP: number };
  riasec: {
    scores: Record<'R' | 'I' | 'A' | 'S' | 'E' | 'C', number>;
    code: string;
    consistency: 'high' | 'medium' | 'low';
    differentiation: 'high' | 'medium' | 'low';
    clarity: 'high' | 'medium' | 'low';
  };
  readiness?: {
    overall: number;
    subScores: Record<string, number>;
  };
  consistencyFlags: string[];
  candidateFields: string[];
}
```

### 9.3 AI 报告请求

```typescript
interface ReportRequest {
  level: Level;
  summary: ScoreResult;
}
```

发送到 AI 服务前，服务端必须从 `ScoreResult` 构造白名单字段，不直接转发客户端提交的对象。

## 10. MVP 功能范围

### 10.1 包含

- 教育阶段选择；
- 四个层级的完整答题流程；
- 五种计分题型；
- 回退修改和提交确认；
- 本地规则评分；
- 图表和文字基础报告；
- 用户自主选择是否启用 AI 解读；
- AI 失败时的本地报告降级；
- 报告导出 PDF；
- 手机端优先的响应式体验。

### 10.2 不包含

- 账号、登录和历史记录；
- 数据库和用户档案；
- 用户样本收集和常模建设；
- 开放文本题；
- 付费和会员；
- 社交分享或排行榜；
- 后台管理；
- 学校或机构批量测评；
- 基于测评结果的广告或营销画像。

## 11. 开发前验收清单

进入页面开发前，题库数据化必须通过以下自动检查：

- L1、L2、L3、L4 分别恰好为 32、46、53、59 题；
- 所有题目 ID 全局唯一；
- 不存在开放文本题；
- 每道题的类型、层级、维度和计分规则完整；
- 排序题的选项数量和计分名次数匹配；
- 迫选题不能将同一选项同时选为最喜欢和最不喜欢；
- 每个展示子维度都存在有效的理论最低分和最高分；
- L1 不包含职业准备度题；
- AI 报告请求不包含逐题答案或身份字段；
- 所有报告页面包含非诊断和非决策依据提示。

## 12. 待后续独立评估的事项

以下事项不属于当前 MVP，不得在开发过程中顺带加入：

- 用户样本收集；
- 人群常模和百分位；
- 信度、效度及学术验证；
- 监护人数字化授权流程；
- 账号与长期报告存储；
- 研究用途的数据匿名化方案；
- 面向学校或机构的批量应用。

其中任何一项启动前，都需要单独完成产品必要性、测量方法、隐私合规和技术安全评估。
