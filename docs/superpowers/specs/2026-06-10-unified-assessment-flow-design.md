# 统一测评流程接入设计

> 日期：2026-06-10
> 状态：已确认

## 目标

把现有测评界面接入统一的 v2 题库和新计分核心，使 L1、L2、L3、L4 都能完成答题并查看基础报告，同时不把答案写入浏览器存储。

本阶段不接入 AI、不收集样本、不建立用户档案，也不增加注册、登录或服务端数据库。

## 数据源

`question-bank-v2.md` 是唯一人工维护题库，生成文件为：

```text
app/src/data/questions/generated.ts
```

运行时不得再读取手写的 L1-L4 题库。现有手写题库文件暂时保留，避免删除未确认的工作区改动，但从运行入口移除。

四个等级的题量固定为：

- L1：32 题；
- L2：46 题；
- L3：53 题；
- L4：59 题。

## 接入方式

新增一个轻量适配层，把统一题目模型转换成现有答题组件需要的展示模型：

- `single` 根据选项数量显示为二选一或场景选择；
- `likert` 使用 1-5 量表组件；
- `ranking` 使用排序组件；
- `forcedChoice` 使用最喜欢/最不喜欢组件。

适配层只负责展示格式，不复制题目文本或评分规则。

首页、测试页和等级元数据继续复用现有界面结构。四个等级都标记为可用，并显示统一题库中的实际题量。

## 答题与结果流转

答题期间，答案只保存在 `QuestionEngine` 的 React 内存状态中。

提交时：

1. 使用新计分核心在当前页面完成计分；
2. 把不含原始答案的报告结果写入内存中的结果上下文；
3. 跳转到结果页；
4. 结果页从上下文读取报告并展示。

不得使用：

- `sessionStorage`；
- `localStorage`；
- IndexedDB；
- Cookie；
- URL 查询参数承载答案或完整报告。

用户刷新结果页或直接访问结果页时，内存结果会消失。页面应提示“没有找到本次测评结果，请重新开始”，这是本阶段接受的隐私优先行为。

## 评分与报告转换

新计分核心：

```text
app/src/domain/assessment/score.ts
```

继续负责答案完整性校验、原始分累加、理论范围计算和 0-100 标准化。

新增纯函数把 `AssessmentScore.normalizedScores` 转换为现有报告页使用的展示结构，包括：

- 六个 RIASEC 分数和前三位代码；
- 荣格 EI、SN、TF、JP 四条光谱；
- 按命名空间分组的维度分数；
- L2-L4 的发展准备度；
- 基础探索方向建议。

报告转换不得重新计算题目分数，也不得依赖旧手写题库。

## 组件边界

```text
app/src/
  data/questions/generated.ts       唯一运行题库
  domain/assessment/score.ts        基础计分
  domain/assessment/report.ts       报告展示模型转换
  lib/questions/index.ts            题库与等级元数据适配
  lib/questions/types.ts            现有界面所需展示类型
  components/test/QuestionEngine.tsx
                                     内存答题、提交计分
  components/assessment/ResultStore.tsx
                                     仅当前页面会话的结果上下文
  app/layout.tsx                    挂载结果上下文
  app/result/page.tsx               读取内存结果
```

如现有组件可以直接接受统一题目类型，则不额外复制类型；适配应保持在最小范围内。

## 错误处理

- 非法等级返回首页提示；
- 题库为空时不允许开始；
- 答案未完整时不允许提交；
- 计分失败时留在测评页并显示简短错误；
- 结果上下文为空时提示重新测评；
- 不把答案或报告内容打印到浏览器日志。

## 测试与验收

自动测试必须证明：

- 四个等级都从 `generated.ts` 读取，题量分别为 32、46、53、59；
- 四种统一题型都能转换为现有答题组件可用的格式；
- 每个等级的完整模拟答案都能生成报告展示模型；
- 报告中的 RIASEC 和荣格值都在 0-100；
- L1 不输出准备度，L2-L4 输出准备度；
- 项目源码不再引用 `sessionStorage` 或 `localStorage`；
- 直接进入结果页会显示无结果提示。

最终运行：

```bash
npm test
npm run lint
npm run build
```

并在浏览器中分别打开 L1-L4 的第一题，完成一套可快速自动作答的流程验收。

## 非目标

本阶段不实现：

- AI 文字解读；
- PDF 导出；
- 分享链接；
- 样本收集和统计；
- 用户身份信息；
- 跨刷新恢复答题进度；
- 正式职业推荐算法或专业诊断。
