# 项目初始化与题库数据化设计

> 日期：2026-06-09
> 状态：已确认

## 目标

完成可运行的 Next.js + TypeScript 项目基础，将 `question-bank-v2.md` 转换为类型安全的题库数据，并用最小评分引擎证明四套题库可以被完整读取和计分。

本阶段不制作正式产品页面，不接入 AI，也不实现 PDF。

## 技术方案

- Next.js App Router
- React
- TypeScript 严格模式
- Tailwind CSS
- Vitest
- Zod
- Node.js 题库构建脚本

保留 `question-bank-v2.md` 作为唯一人工维护题库。构建脚本读取 Markdown，生成 TypeScript 数据文件供应用和测试直接使用。

## 文件边界

```text
src/
  app/                       Next.js 最小入口
  domain/assessment/
    types.ts                 题目、答案和评分类型
    score.ts                 基础计分和标准化
    score.test.ts            评分行为测试
  data/questions/
    generated.ts             自动生成的四层题库
    generated.test.ts        题量和完整性测试
scripts/
  generate-question-data.mjs Markdown 到 TypeScript
```

`generated.ts` 不手工编辑。Markdown 更新后必须重新运行生成命令。

## 题目数据模型

每道题包含：

- 稳定 ID；
- 教育阶段；
- 最终显示序号；
- 题型；
- 题目文本；
- 选项；
- 评分目标；
- 量表题的直接评分目标；
- 原始 Markdown 片段，用于追溯。

支持四种答案结构：

- 单选；
- 1–5 量表；
- 排序；
- 最喜欢/最不喜欢迫选。

## 评分键标准化

所有题库标注转换为机器可识别的评分键：

```text
energy.*
intelligence.*
riasec.R | I | A | S | E | C
drive.*
jung.E | I | S | N | T | F | J | P
readiness.*
```

复合标注拆成多个评分目标。例如：

```text
[语言智能+人际智能] +2
```

转换为语言智能和人际智能各 `+2`。

以下标注计为零分，不创建评分目标：

- 平衡；
- 其他方向；
- 不计分；
- 仅用于报告分流但没有明确量化目标的说明。

无法识别且不是上述零分标注的内容必须导致生成失败，不允许静默忽略。

## 题型转换

- 二选一和场景选择转换为 `single`。
- 量表题转换为 `likert`。
- 排序题转换为 `ranking`，使用 `5/3/1/0`。
- 迫选题转换为 `forcedChoice`，默认最喜欢 `+4`、最不喜欢 `-2`。
- 仅要求单选的迫选题转换为 `single`，使用题目原标注分值。

反向量表题保留 `reverse: true`，计分时将 1–5 转换为 5–1。

## 基础评分

评分引擎接收一个层级和完整答案：

1. 验证问题属于所选层级；
2. 验证答案类型与题型匹配；
3. 累加各机器评分键的原始分；
4. 根据该层级题库中每个评分键的理论最低分和最高分转换为 0–100；
5. 返回原始分与标准化分。

本阶段只证明基础计分链路可运行，不实现：

- RIASEC 一致性、区分度和明确度阈值；
- 荣格四条光谱的组合解释；
- 职业方向映射；
- AI 报告。

## 测试与验收

自动测试必须证明：

- L1/L2/L3/L4 分别为 32/46/53/59 题；
- 190 个题目 ID 全局唯一；
- 不存在开放题；
- 所有题型和评分键均可识别；
- 所有选项 ID 在题内唯一；
- 复合评分标注会给每个目标完整加分；
- 反向量表正确反向；
- 排序题按 5/3/1/0 计分；
- 迫选题按最喜欢和最不喜欢分别计分；
- 一份程序生成的完整模拟答案能够输出全部基础标准化分数；
- `npm test`、`npm run lint` 和 `npm run build` 通过。
